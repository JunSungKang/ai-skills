# 프로젝트 검토 체크리스트 상세 참조

## 1. 요구사항 명세 검토

### 완전성 체크
- [ ] 모든 이해관계자의 요구사항이 수집되었는가?
- [ ] 기능 요구사항과 비기능 요구사항이 모두 정의되었는가?
- [ ] 제약 사항(기술, 예산, 일정)이 명확히 기술되었는가?
- [ ] 가정 사항과 의존성이 명시되었는가?

### 명확성 체크
- [ ] 요구사항이 모호하지 않고 명확하게 기술되었는가?
- [ ] 측정 가능한 성능 기준이 포함되었는가? (예: "빠르게" → "응답시간 2초 이내")
- [ ] 예외 케이스와 엣지 케이스가 고려되었는가?

### 추적 가능성 체크
- [ ] 각 요구사항에 고유 ID가 부여되었는가?
- [ ] 요구사항 → 기능명세 → 개발 → 테스트 추적이 가능한가?
- [ ] 우선순위가 명확히 설정되었는가? (MoSCoW 또는 수치)

---

## 2. 기능명세 검토

### 상세도 체크
- [ ] 각 기능의 입력/처리/출력이 명확히 정의되었는가?
- [ ] UI/UX 흐름이 와이어프레임 또는 텍스트로 기술되었는가?
- [ ] API 명세 (엔드포인트, 파라미터, 응답)가 정의되었는가?
- [ ] 에러 케이스와 처리 방법이 명시되었는가?

### 일관성 체크
- [ ] 용어가 문서 전체에서 일관되게 사용되었는가?
- [ ] 다른 기능과의 연계/의존성이 명확한가?
- [ ] 버전 이력이 관리되고 있는가?

### 테스트 가능성 체크
- [ ] 각 기능이 테스트 케이스로 변환 가능한가?
- [ ] 인수 조건(Acceptance Criteria)이 명확한가?

---

## 3. 개발 산출물 검토

### 코드 품질
- [ ] 코딩 컨벤션이 일관되게 적용되었는가?
- [ ] 코드 중복이 최소화되었는가?
- [ ] 하드코딩된 값이 없는가? (매직 넘버, URL, 패스워드)
- [ ] 에러 처리가 적절히 구현되었는가?
- [ ] 로그가 적절히 출력되는가? (너무 많지도, 적지도 않게)

### 아키텍처
- [ ] 설계 원칙이 지켜졌는가? (SOLID, DRY, KISS)
- [ ] 레이어 분리가 명확한가?
- [ ] 확장성을 고려한 설계인가?

### 문서화
- [ ] README가 최신 상태로 유지되는가?
- [ ] API 문서가 코드와 일치하는가?
- [ ] 복잡한 비즈니스 로직에 주석이 있는가?
- [ ] 배포/운영 가이드가 존재하는가?

### 테스트
- [ ] 단위 테스트가 작성되었는가? (커버리지 목표 달성)
- [ ] 통합 테스트가 작성되었는가?
- [ ] 테스트가 자동화되었는가? (CI/CD 연동)

---

## 4. UI/퍼블리싱 검토

### UX 흐름
- [ ] 사용자 시나리오에 따른 흐름이 자연스러운가?
- [ ] 오류/경고 메시지가 사용자 친화적인가?
- [ ] 로딩 상태, 빈 상태, 에러 상태 UI가 구현되었는가?
- [ ] 주요 액션의 피드백이 즉각적인가?

### 접근성 (Accessibility)
- [ ] 이미지에 대체 텍스트(alt)가 있는가?
- [ ] 키보드만으로 전체 기능 사용이 가능한가?
- [ ] 색상 대비비가 충분한가? (4.5:1 이상)
- [ ] ARIA 레이블이 적절히 사용되었는가?

### 반응형 & 호환성
- [ ] 모바일, 태블릿, 데스크탑에서 정상 동작하는가?
- [ ] 주요 브라우저에서 테스트되었는가? (Chrome, Firefox, Safari, Edge)
- [ ] 해상도별 레이아웃 깨짐이 없는가?

### 브랜드 일관성
- [ ] 디자인 시스템/가이드라인을 준수하는가?
- [ ] 색상, 폰트, 간격이 일관되게 적용되었는가?
- [ ] 아이콘/이미지가 일관된 스타일인가?

---

## 5. 일정 & 리소스 검토

### 일정 관리
- [ ] 마일스톤과 데드라인이 현실적으로 설정되었는가?
- [ ] 임계 경로(Critical Path)가 파악되고 있는가?
- [ ] 버퍼(여유 시간)가 충분히 확보되었는가?
- [ ] 현재 진행률이 계획과 일치하는가?

### 리스크 관리
- [ ] 주요 리스크가 식별되고 문서화되었는가?
- [ ] 각 리스크에 대한 대응 방안이 마련되었는가?
- [ ] 리스크 오너(담당자)가 지정되었는가?

### 의사소통
- [ ] 이해관계자와의 정기 보고 채널이 있는가?
- [ ] 변경 사항 관리 프로세스가 있는가?
- [ ] 이슈 추적 시스템이 운영되는가?
