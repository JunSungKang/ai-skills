# 정보보안 평가 기준 상세 참조

## ISMS-P (정보보호 및 개인정보보호 관리체계)

### 관리 체계 수립 및 운영 (16개 통제)
- 1.1 경영진의 참여
- 1.2 최고책임자의 지정
- 1.3 조직 구성
- 1.4 범위 설정
- 1.5 정책 수립
- 1.6 자원 할당

### 보호 대책 요구사항 (64개 통제)
**2.1 정책, 조직, 자산 관리**
**2.2 인적 보안**
**2.3 외부자 보안**
**2.4 물리 보안**
**2.5 인증 및 권한관리**
- 패스워드 정책, MFA, 최소 권한 원칙

**2.6 접근통제**
- 네트워크 접근통제, 시스템 접근통제

**2.7 암호화 적용**
- 전송 구간 암호화 (TLS 1.2+)
- 저장 데이터 암호화 (AES-256)
- 개인정보 암호화 (비밀번호 단방향 해시)

**2.8 정보시스템 도입 및 개발 보안**
- 보안 요구사항 정의
- 시큐어 코딩
- 취약점 점검

**2.9 시스템 및 서비스 운영관리**
**2.10 시스템 및 서비스 보안관리**
**2.11 사고 예방 및 대응**
**2.12 재해복구**

### 개인정보 처리 단계별 요구사항 (22개 통제)
- 3.1 개인정보 수집 시 보호 조치
- 3.2 개인정보 보유 및 이용 시 보호 조치
- 3.3 개인정보 제공 시 보호 조치
- 3.4 개인정보 파기 시 보호 조치
- 3.5 정보주체 권리 보호

## CSAP (클라우드 서비스 보안인증)

### IaaS 기준 (표준 등급)
- 물리적 보안 (데이터센터)
- 네트워크 보안 (방화벽, IDS/IPS)
- 가상화 보안 (VM 격리)
- 데이터 보안 (암호화, 백업)
- 운영 보안 (패치, 로그)

## 관련 법령

### 개인정보보호법 (2023년 개정)
- 제24조: 고유식별정보 처리 제한
- 제25조: 영상정보처리기기 설치·운영 제한
- 제29조: 안전조치 의무
- 제34조: 개인정보 유출 통지 (72시간 이내)
- 제39조: 손해배상 책임

### 정보통신망법
- 제45조: 정보통신망 침해행위 금지
- 제45조의3: 정보보호 최고책임자 지정

### 전자금융거래법 (금융권)
- 제21조: 안전성 확보 의무

## 보안 검토 체크리스트

### 인증 & 인가
- [ ] 강력한 패스워드 정책 (8자 이상, 복잡도)
- [ ] MFA 적용 여부
- [ ] 세션 타임아웃 설정
- [ ] JWT 만료 시간 및 갱신 정책
- [ ] 최소 권한 원칙 적용

### 암호화
- [ ] 전송 구간 TLS 1.2+ 적용
- [ ] 저장 데이터 암호화 (개인정보, 민감정보)
- [ ] 패스워드 단방향 해시 (bcrypt, argon2)
- [ ] 암호키 관리 정책

### 입력 검증
- [ ] SQL Injection 방지 (Prepared Statement)
- [ ] XSS 방지 (출력 인코딩)
- [ ] 파일 업로드 검증 (확장자, MIME, 용량)
- [ ] API 입력값 검증

### 로깅 & 모니터링
- [ ] 보안 이벤트 로그 (로그인 성공/실패, 권한 변경)
- [ ] 로그 무결성 보장
- [ ] 개인정보 로그 마스킹
- [ ] 침해 탐지 모니터링
